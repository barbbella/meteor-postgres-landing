Database Everywhere

We implemented PostgresSQL on the back-end, and used alasql to implement minisql (our version of minimongo) on the front-end. We designed an ORM modeled after Ruby's Active Record to wrap both allowing for one interface to interact with both. This also aligns our implementation with Meteor's principle of 'One Language'.

Full Stack Reactivity

Using Postgres' Notify capabilities we were able to have updates to the server automatically propagate to connected clients using DDP.

Latency Compensation

In our implementation of SQL.Collection we transact with the local database, minisql, before we invoke the server side transaction. When the client receives the notification from the client of the new transaction, it updates the local database to ensure correct information.
